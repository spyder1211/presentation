プロンプト作成の基本フレームワーク

📝 効果的なプロンプトの5要素

あなたは[役割・専門家]です。 ← 要素1: 役割設定
私は[依頼者の立場や目的]です。 ← 要素2: 文脈設定
以下の条件を守って出力してください。

## 出力条件
1. 目的： [最終的に達成したいことを明記] ← 要素3: 目的
2. 形式： [箇条書き / 表形式 / 文章形式など] ← 要素4: 形式
3. 文体： [簡潔でわかりやすい / 専門的で正確など] ← 要素5: 文体
4. 制約： [字数制限 / 難しい用語には注釈を入れるなど]
5. 出力の質： 
   - 論理的で一貫性があること
   - 冗長さを避け、要点を押さえること

✅ このフレームワークのメリット

  • 再現性が高い：誰でも同じ品質の出力を得られる
  • カスタマイズが容易：業務に応じて条件を調整可能
  • 品質管理が可能：出力条件で品質基準を明確化

実践事例①：メール対応の効率化

📧 市民からのメール対応を自動化

❌ 改善前：シンプルなプロンプト

このメールの返信を作成してください。
[メール本文]

問題点：文体が不統一、必要情報の欠落、公的機関らしさの不足

✅ 改善後：構造化されたプロンプト

あなたは地方自治体の職員です。
市民から届いたメールに対して、
誠実かつ丁寧で、公的機関らしい
返信文を作成してください。

【条件】
- 冒頭で感謝を述べること
- 要望内容を要約して触れること
- 現状の説明や対応方針を示すこと
- 敬語でまとめること

📊 導入効果

  • 返信作成時間：15分 → 3分（80%削減）
  • 品質の標準化：新人職員でもベテラン並みの文面作成可能
  • ミスの削減：必要情報の記載漏れがゼロに

実践事例②：議事録の有効活用

📚 議事録から価値ある情報を抽出

活用例1：市民向けFAQの自動生成

あなたは自治体の広報担当ライターです。
議事録から子育てに関するFAQを作成してください。

## 出力条件
1. Q&A形式（Markdown見出しつき）
2. 親しみやすく、誰でもわかる言葉で
3. 1つの質問は100文字以内
4. 回答は200文字以内

活用例2：各担当課のTODOリスト自動作成

議事録から各担当課の持ち帰り課題を抽出してください。

## 抽出対象
- 「確認する」「検討する」と答えた事項
- 「対応を依頼された」事項
- 「現状不明」「把握していない」と答えた事項

💡 活用のポイント

  • 議事録の死蔵を防ぐ：膨大な議事録から必要情報を瞬時に抽出
  • 業務の見える化：課題や宿題事項を自動的にリスト化
  • 市民サービスの向上：議事録から市民向け情報を自動生成

議事録活用の実績

📈 令和7年3月都留市社会厚生常任委員会議事録から

FAQ自動生成の出力結果

FAQ 15項目を5分で作成（手作業なら2時間）

TODO抽出の出力結果

TODO 23項目を自動抽出（見落としゼロ）

実践事例③：画像処理の自動化

🖼️ レシート画像から経費内訳を自動作成

プロンプト例

あなたは経理担当者です。
レシート画像を確認し、経費内訳を作成してください。

## 出力条件
1. 出力形式はMarkdown
2. 日付、店舗名、品目、金額を表形式で整理
3. 金額は全て数値として扱い、合計金額も算出
4. 不明な項目は「不明」と記載

入力：レシート画像

出力：自動生成された経費内訳表

🚀 導入メリット

  • 入力作業の削減：手入力不要で転記ミスゼロ
  • 処理時間の短縮：10枚のレシートを3分で処理
  • 統一フォーマット：誰が処理しても同じ形式

プロンプト改善のプロテクニック

🔧 AIにプロンプト自体を改善してもらう

改善依頼の例

地方自治体の職員としてメールを返信したいです。

[メール本文]

このプロンプトをもっとよくして。

AIが提案する改善版

あなたは地方自治体の職員です。
市民から届いたメールに対して、
以下の条件で返信文を作成してください。

【条件】
1. 冒頭で感謝を述べる
2. 要望内容を簡潔に要約
3. 現状の説明や対応方針を説明
4. 誠実で丁寧なトーン
5. 1文は60文字程度
6. 連絡先や追加問い合わせへの案内

📊 改善のポイント

  1. 具体性を高める：曖昧な指示を具体的な条件に分解
  2. 構造化する：情報を階層的に整理
  3. 出力形式を明確化：期待する結果を具体的に示す
  4. 制約条件を追加：文字数、文体、必須要素を明記
